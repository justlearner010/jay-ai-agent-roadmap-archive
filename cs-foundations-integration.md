# 科班基础嵌入方案

核心原则：不要把“科班基础”和“AI Agent 求职”拆成两条互相竞争的路线。用 AI Agent 项目作为主线，把科班基础嵌入项目中学。

## 总体比例

每周 20 小时建议这样分配：

- 10 小时：AI Agent / RAG 项目。
- 4 小时：Python / Java + 后端工程。
- 4 小时：科班基础。
- 2 小时：英文阅读、JD 分析、简历和复盘。

这 4 小时科班基础不要泛学。每周只围绕当前项目最需要的基础知识补一小块。

Java 后端副线只占整体约 30%。它的作用是把 AI 项目包装成更接近真实业务系统的后端服务，不是把主线改成普通 Java 培训。

## 每周选择规则

每周日只问 3 个问题：

1. 这个星期的项目卡在哪里？
2. 这个卡点背后对应哪门科班基础？
3. 我能不能用 4 小时学到足够解决它的程度？

如果答案是“能帮助当前项目跑通、变稳、变快、变清楚”，就学。  
如果只是“看起来很正统但短期用不上”，先放后面。

## 阶段嵌入表

### 阶段 1：AI 学习助手

时间：2026-05 到 2026-07

项目主线：

- AI 学习助手。
- 文本输入、总结、提问、历史记录、基础 API。

嵌入的科班基础：

- 数据结构：数组、字符串、哈希表、栈、队列。
- 工程基础：Git、命令行、文件系统、调试。
- Python 基础：函数、类、异常、测试。
- 第 5 周后加入 Java 对照：集合、类、异常、Maven、JUnit。

学习方式：

- 做文本统计、词频统计、文本切块、JSON 保存、CLI 参数解析。
- 每个小功能都写 1-2 个测试。
- 少量用 Java 重写已经理解的核心练习，例如词频统计或文本校验；只做对照，不重复刷两套题。
- 每周至少一次 Git commit，并写清楚 commit message。

### 阶段 2：个人 RAG 知识库

时间：2026-08 到 2026-10

项目主线：

- 文档导入、文本切块、embedding、向量检索、带来源回答。

嵌入的科班基础：

- 数据库：SQL、表设计、索引、事务、PostgreSQL。
- 计算机网络：HTTP、状态码、CORS、DNS、HTTPS。
- 数据结构：树、堆、排序、二分、简单图。
- Java 后端：Spring Boot REST API、参数校验、异常处理、Repository / Mapper、数据库事务。

学习方式：

- 用数据库保存文档 metadata、会话、问题和回答。
- 用 HTTP API 暴露导入、检索、问答接口。
- 用 Java Spring Boot 做业务后端或 API Gateway，管理用户、文档、任务和查询历史。
- 用 Python / FastAPI 做 RAG 推理服务，负责切块、embedding、检索和生成。
- 用小型评估集比较不同 chunk size 和检索参数。

### 阶段 3：LangGraph 研究 Agent

时间：2026-11 到 2027-01

项目主线：

- 状态机、工具调用、任务拆解、执行日志、失败重试、人工确认。

嵌入的科班基础：

- 操作系统：进程、线程、并发、文件、环境变量。
- 后端系统：后台任务、日志、超时、重试、权限、任务状态、工具调用记录。
- 算法：图搜索、BFS/DFS、动态规划入门。
- Java 后端：用 Spring Boot 或 FastAPI 暴露任务创建、状态查询、执行日志查询接口。

学习方式：

- 把 Agent 的执行过程看成状态图。
- 为每个工具设置超时、白名单和错误处理。
- 用 JSONL 或数据库保存执行轨迹。
- 让执行轨迹可以通过后端 API 查询，用它支撑面试中的系统设计讲解。

### 阶段 4：作品集与面试

时间：2027-02 到 2027-04

项目主线：

- 项目打磨、部署、README、demo、简历、面试。

嵌入的科班基础：

- 算法刷题：数组、哈希、双指针、栈、队列、树、图、动态规划基础。
- 系统设计入门：API 设计、数据库设计、缓存、队列、日志、监控、异常处理。
- Java + Spring Boot 强化：REST API、数据库 CRUD、单元测试、Redis 基础、AI 服务调用。

学习方式：

- 每周固定刷 5-8 道基础算法题。
- 把项目里的 API、数据库表、日志、缓存、队列设计讲成面试故事。
- 准备两版项目讲法：AI Agent 应用后端版、Java 后端版。
- 根据 JD 决定是否继续加强 MyBatis / JPA、Redis 或 Spring Boot 测试。

## 当前不用深挖

这些不是不重要，而是暂时不用抢主线时间：

- 编译原理。
- 计算机组成原理深度细节。
- 分布式一致性。
- Kubernetes。
- JVM 调优。
- 复杂微服务治理。
- Spring Cloud 深入。
- 高并发深入。
- 大规模机器学习训练。

## 验收标准

每个月检查一次：

- 我是否能用项目例子解释一个科班概念？
- 我的项目是否因为这个基础知识变得更稳定、更快或更清楚？
- 我是否能把这个基础知识写进 README、面试讲解或简历 bullet？

如果一个知识点学完后无法回答这 3 个问题，说明它暂时学得太散，应该回到项目主线。
