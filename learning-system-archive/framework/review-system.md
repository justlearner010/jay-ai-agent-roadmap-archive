# 学习诊断与周验收系统

目标：反馈服务于项目问题和能力增长，不再用每日完成率推动学习。第 1-4 周已有的每日作业评分和 `homework-reviews/` 保留为历史记录；从第 5 周开始使用周中诊断和周末验收。

## 当前反馈节奏

### 周中诊断：不评分

1. 展示当前可以运行的结果，失败结果同样有效。
2. 写出本周主问题和当前最大卡点。
3. Codex 检查问题是否过大、证据是否有效、任务池是否需要调整。
4. 只给出一个优先实验和最多两个风险提示，不生成补作业清单。
5. 将结果保存到 `weekly-reviews/YYYY-Wxx-midweek.md`。

周中诊断使用 `templates/midweek-diagnosis.md`，不得出现百分制、完成任务卡数量或“补齐原日程”的要求。

### 周末验收：按结果做决策

1. 回答本周主问题。
2. 展示可运行功能或明确实验结论。
3. 提供至少两个成功或失败证据。
4. 解释一项关键实现、一次失败及其原因。
5. 选择继续、转向或暂存，并写明理由。
6. 将结果保存到 `weekly-reviews/YYYY-Wxx-acceptance.md`，同步更新周追踪表。

周末验收使用 `templates/weekly-acceptance.md`。验收只标记“通过 / 部分通过 / 未通过”，不用百分制。

## 诊断范围

- 当前项目代码与实际运行结果。
- 测试、类型检查、构建、截图或失败日志。
- 本周主问题、任务池和范围边界。
- 是否能用自己的话解释关键逻辑。
- 相邻自由探索是否实际发生，但不评价它是否产出代码。

## 验收判断

- **通过**：主问题得到证据支持的回答，可演示结果达到验收条件，能够解释关键实现和失败。
- **部分通过**：产生了有效证据并显著缩小问题，但可演示闭环或解释仍不完整。
- **未通过**：没有真实运行或失败证据，无法判断实验是否减少不确定性。

未通过不产生逐日补债。下一周应缩小问题、改变实验或暂存方向；只有影响正确性、安全性和可运行性的项目问题需要优先处理。

## Codex 的边界

- 你先提出最想解决的周问题，Codex帮助收敛，不代替选题。
- Codex可以指出风险、建议实验和审查证据，不按预写任务完成率评价。
- 默认只给诊断意见，不直接替你重写学习成果；你明确要求修改文件时才实施。
- AI生成代码只有在你能解释、验证并完成一次修改后，才计入个人资产。

## 历史兼容

- `homework-reviews/` 和 `trackers/homework-review-log.csv` 冻结为第 1-4 周历史，不删除、不重写。
- 旧的 `templates/homework-review-template.md` 和 `templates/daily-homework-submission-template.md` 保留供查阅，不再作为当前入口。
- 新反馈统一进入 `weekly-reviews/`，周级状态写入 `trackers/weekly-log.csv`。

## 推荐请求格式

周中：

```text
Codex，做本周诊断，不评分。
本周主问题：...
当前可运行结果：...
最大卡点：...
已有证据：...
```

周末：

```text
Codex，做本周验收。
本周主问题和回答：...
演示结果：...
证据：...
关键失败：...
我的下周决策：继续 / 转向 / 暂存
```
