# 项目 2：LangGraph 研究 Agent

## 目标

做一个可以接收研究问题、拆解任务、收集信息并生成结构化报告的 Agent。重点不是“聊天”，而是能展示清楚的执行过程。

## 必做功能

- 用 LangGraph 搭建状态机。
- 至少包含规划、工具调用、综合整理、最终报告几个节点。
- 至少接入两个工具，例如网页搜索、本地文件检索、笔记写入。
- 每个节点和工具调用都保存执行日志。
- 工具失败时有重试或降级处理。
- 对高成本或高风险动作加入人工确认。

## 推荐技术栈

- Python
- LangGraph
- v1 可以先做 CLI，之后再接 FastAPI
- SQLite 或 JSONL 保存日志
- pytest 测试节点和工具行为

## 验收标准

- Agent 能完成 5 个真实研究任务。
- 日志能展示状态转换和工具输出。
- 最终报告包含来源或证据。
- README 写清楚失败案例。
- README 包含架构图和一段 demo 执行记录。
