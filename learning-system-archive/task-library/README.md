# 学习任务库与历史作业

第 1-4 周文件是已经发生的每日作业历史。第 5-12 周文件已经转换为按项目卡点调用的任务卡：保留知识讲解、实验、提示和验收，但不再代表日期、必做顺序或学习债务。

当前执行入口是仓库根目录的 `this-week.md`。你先提出本周主问题，再根据项目当前未知点选择任务卡；没有对应卡点时，不需要打开或完成任务卡。

普通任务卡包含：

- 今日目标
- 作业要求
- 推荐阅读材料
- 提交物
- 验收标准
- 批改重点

启发式任务卡额外包含：

- 昨日无提示回忆
- 真实问题和学习原因
- 初始预测与最小知识讲解
- 跟做实验与核心动手任务
- 迁移挑战与三级折叠提示
- 内化检查和提示等级记录

启发式任务可配合 `templates/task-guided-study-note-template.md` 边学边记。它不会替代简短的每日学习日志；当天结束时只需从笔记中提取结论写入日志。

使用方式：

1. 打开 `this-week.md`，确认本周主问题和当前最大未知点。
2. 在对应阶段 README 的分类索引中选择一张能减少该未知点的任务卡。
3. 启发式任务先预测，再阅读最小讲解并完成一个实验；卡住时按提示 1、2、3 展开。
4. 只做到足以推进本周问题的深度，不因为打开任务卡就必须完成全部章节。
5. 保存成功或失败证据，写下下一次最自然的动作；未选择和未完成的任务不需要补。
6. 周中使用 `templates/midweek-diagnosis.md` 做无评分诊断，周末使用 `templates/weekly-acceptance.md` 验收。

## 执行原则

- 第 5 周启发式任务按“讲解和理解约 40%，编码、验证和迁移约 60%”设计，但只在对应卡点出现时选用。
- 第 5 周是首轮试运行；周末根据项目证据决定继续、转向或暂存。
- 任务中的折叠提示必须逐级使用，不提供可直接复制的完整核心作业答案。

- 每周只追一个主问题，不把后期 RAG/Agent 标准提前压到当前阶段。
- 每次学习优先完成最小实验：一个假设、一个动作、一个证据。
- 第 1-4 周不加 Java，先跑通 Python、pytest、CLI 和 GitHub 证据链。
- 第 5-8 周每周只安排 2 个 Java 副线触点，目标是 Java 基础、Maven、JUnit 和最小 Spring Boot API。
- 第 9-12 周每周安排 3 个 Java 后端触点，目标是 CRUD、数据库、任务模型和 Java + AI 服务调用雏形。
- Java 副线不要求每次推进；主线实验吃紧时，优先保证 AI 学习助手形成可验收闭环。
- Java 副线必须从已有 Python / AI 学习助手任务过渡：优先重写、封装、测试或网关化已经理解的文本处理、校验、API、持久化和任务记录能力。
- 科班基础嵌入使用 `resources/cs-foundations-side-track.md` 的小格：基础点、对应项目任务、最小验收。
- 算法按项目卡点或独立面试准备需要进行，不再作为每日任务。
- 每周固定保留 4 小时相邻自由探索，不因主线延期取消，也不要求当周代码产出。
- 连续两次实验没有减少不确定性时，缩小问题、改变实验或在周中诊断中调整任务池。
- 每周至少留下两个可验收证据：代码、测试、日志、截图、demo 或失败记录。
- 推荐阅读材料放在作业要求下方，是为了边做边查；如果阅读和作业冲突，优先完成作业。
- 周末必须验收并更新 `trackers/weekly-log.csv`，记录继续、转向或暂存。
