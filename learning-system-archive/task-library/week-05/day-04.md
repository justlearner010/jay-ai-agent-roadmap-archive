## 4. 为什么学

React state 用来保存会随交互变化的数据；输入变化后更新 state，React 再根据新 state 重新计算页面。

## 5. 最小知识讲解

### 5.1 状态变化链

```text
用户输入 -> onChange 事件 -> 更新 state -> React 重新渲染 -> UI 显示新值
```

`useState` 返回当前值和更新它的函数。受控输入把 textarea 的 `value` 绑定到 state，并在 `onChange` 中更新该 state。

### 5.2 派生值不一定需要额外 state

字符数可以从当前文本 `text.length` 计算。如果一个值可以由现有 state 直接算出，通常先不要保存第二份状态，避免两份数据不同步。

### 5.3 空白输入

`text.length > 0` 不能拒绝三个空格。`trim()` 可以生成去掉首尾空白后的字符串，再判断是否为空。

与 Python 对照：这类似先读取字符串，再使用 `strip()` 判断是否真正包含内容。

常见误区：state 更新用于触发下一次渲染，不要把它理解成普通变量的原地修改。
