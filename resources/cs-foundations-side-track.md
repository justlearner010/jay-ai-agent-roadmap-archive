# 科班基础嵌入过渡资源

这个文件把科班基础写成和 Java 副线一样可执行的格子。它不是新增一条重学习线，而是帮助你在每天的项目任务里说清楚：今天练到的计算机基础是什么、它对应哪个项目动作、最小验收是什么。

## 使用原则

- 每次只抓一个基础点，不在当天额外开大课。
- 基础点必须绑定项目任务：代码、测试、接口、数据模型、日志、README 或复盘。
- 最小验收必须能被一句话解释、一个测试、一个图或一段 README 证明。
- 如果当天主线任务已经超时，科班基础只保留“解释题”，不额外加代码。
- 算法微练习每天最多 20-30 分钟，优先 1 道 Easy。做不完就记录思路和卡点，不能把当天项目任务挤掉。

## 每日算法微练习

当前阶段的算法练习只服务两个目标：保持手感，巩固 Python 数据结构。

- 题型范围：字符串、数组、哈希表、栈、队列。
- 完成标准：通过代码实现，或写出输入输出、解题思路、伪代码和卡点。
- 复盘方式：每周记录做了几道题、卡在哪个数据结构、下周是否需要减少或增加。
- 暂不进入：动态规划、图论难题、Hard 题、竞赛技巧和题海式刷量。

## 每日固定小格

在关键每日作业里使用这个格式：

```md
## 科班基础嵌入

- 今日基础点：
- 对应项目任务：
- 最小验收：
```

## 基础点解释模板

每天写学习日志时，用 3 句话回答：

1. 今天的基础点是什么？
2. 它在项目里对应哪个文件、接口、数据结构或流程？
3. 如果面试官问这个点，我能用哪个项目例子解释？
