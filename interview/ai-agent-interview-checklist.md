# AI Agent 实习面试检查清单

## 必须能讲清楚

- 什么是 RAG，以及为什么它能减少幻觉。
- embedding、向量搜索、关键词搜索的区别。
- chunk size 如何影响检索质量。
- 如何评估一个 RAG 系统。
- 什么是 Agent，什么时候不应该用 Agent。
- LangGraph 如何表示状态和状态转换。
- 工具调用是怎么工作的。
- 如何避免不安全或高成本的工具调用。
- 你的项目如何处理错误和日志。

## 编程练习重点

- Python 列表、字典、类、异常、文件。
- 类型标注和 Pydantic model。
- 异步基础。
- HTTP request/response。
- 简单 SQL 查询。
- 基础数据结构和算法。
- 用 pytest 测试纯函数和 API 逻辑。

## 项目讲解结构

用这个 5 分钟结构：

1. 问题：谁需要这个系统，为什么需要。
2. 架构：前端、后端、LLM、检索/Agent 层、存储。
3. 难点：一个具体技术问题，以及你怎么解决。
4. 评估：你怎么知道它有效。
5. 下一步：如果继续做，你会改进什么。

## 面试中要避免

- 只说“我用了 LangChain”，但讲不清数据流。
- 没有测试或评估。
- 做了一个没有明确场景的聊天机器人。
- 不知道幻觉可能从哪里产生。
- 解释不清自己项目里的代码。
