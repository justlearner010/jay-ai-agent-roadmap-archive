# 每日作业批改系统

目标：让每天的学习形成闭环。不是只“学了”，而是每天都有作业、批改、修改、复查和 GitHub 证据。
说明：默认只在对话里给批改意见，不直接替用户重写作业。只有用户明确要求“帮我修改文件”时，Codex 才可以改文件。这样可以避免用户直接依赖 AI 结果，而是先理解自己的代码。

## 每日流程

1. 当天完成学习任务和代码练习。
2. 用 `templates/daily-learning-log-template.md` 写学习日志。
3. 用 `templates/daily-homework-submission-template.md` 整理作业提交信息。
4. 对 Codex 说：`codex，批改今天作业`。
5. Codex 按 `templates/homework-review-template.md` 批改。
6. Codex 把当天批改结果保存到 `homework-reviews/YYYY-MM-DD-review.md`。
7. 第二天优先修改“必改项”。
8. 修改后再次提交给 Codex 复查。
9. 通过后再推送到 GitHub，或由 Codex 帮你提交并推送。

前 12 周的批改重点见：`12-week-execution-discipline.md`。批改必须按当前学习阶段来，不把后期工程化标准提前压到第一天。

## 批改范围

每天批改这些内容：
- 代码文件。
- 测试或运行结果。
- 学习日志。
- 截图或 demo。
- commit 或准备提交的改动。
- 是否符合本周计划。
- 是否符合当前阶段目标：入门阶段先看能否运行、能否解释、能否修明显错误；后续再逐步看函数化、测试和工程化。

## 评分标准

总分 100 分：

- 当前阶段完成度：30 分。是否完成了今天真正该练的内容。
- 正确性和可运行性：25 分。代码是否能运行，结果是否大体正确。
- 代码可读性：15 分。变量名、缩进、基本结构是否容易理解。
- 学习日志和证据：20 分。是否记录了做了什么、怎么运行、遇到什么问题。
- 明日可执行性：10 分。明天的修改任务是否具体、少量、能完成。

## 分阶段批改标准

### 入门练习阶段

适用：刚学习一个概念的第 1-3 天，例如刚学 Python 文件读取、循环、字符串处理。

合格标准：

- 代码能运行。
- 能解释主要语句。
- 能完成核心功能。
- 能修正明显错误，比如变量名混乱、统计逻辑不准、忘记关闭文件。
- 学习日志能写清楚今天做了什么。

这个阶段不要求：

- 必须拆成多个函数。
- 必须写 pytest。
- 必须写 CLI。
- 必须写 README。
- 必须达到作品集级别结构。

### 小步改进阶段

适用：同一个练习已经能跑，开始做第二轮修改。

合格标准：

- 把 1-2 段明显重复或混乱的逻辑整理清楚。
- 可以开始写一个简单函数。
- 可以补一个边界情况，例如空文件或只有空格的文件。
- 能说明“为什么这样改更清楚”。

### 工程化练习阶段

适用：一个练习已经读懂并能解释，准备放进长期作品集。

合格标准：

- 有清楚的函数。
- 核心逻辑可以被测试。
- 可以逐步加入 pytest、CLI、README。
- 代码结构便于后续复用。

## 批改等级

- 90-100：优秀，可以直接进入下一个任务。
- 75-89：合格，有少量修改项。
- 60-74：勉强合格，第二天必须先修改。
- 60 以下：当天作业没有形成有效闭环，需要重做关键部分。

## 必改项和建议项

批改时分两类：

- 必改项：当前阶段必须解决的问题。入门阶段优先是正确性、可运行性、能否解释。
- 建议项：以后可以逐步提升的问题，例如函数、测试、README、CLI、项目结构。

第二天优先做必改项。建议项可以在本周内逐步补；工程化建议不能当作第二天必须完成的任务。

## 复查规则

复查只看三件事：

1. 必改项是否完成。
2. 是否产生新的问题。
3. 是否可以上传 GitHub。

复查通过后，在 `trackers/homework-review-log.csv` 里更新状态。

## 批改归档规则

每次批改都要在仓库里留下文件，不只保存在聊天记录中。

- 批改文件目录：`homework-reviews/`。
- 文件名格式：`YYYY-MM-DD-review.md`。
- 文件内容使用 `templates/homework-review-template.md`。
- 如果当天已有批改文件，复查内容追加到同一个文件的“复查结果”部分。
- 批改文件后续可以和代码、学习日志一起提交到 GitHub。

## 你每天提交给 Codex 的推荐格式

```text
codex，批改今天作业。
Codex 按 `templates/homework-review-template.md` 批改

今天的代码：
- python-foundations/xxx.py
- python-foundations/tests/test_xxx.py

学习日志：
- learning-log/YYYY-MM-DD.md

我希望你重点看：
- 代码是否正确
- 是否适合写测试
- 明天应该怎么改
```
