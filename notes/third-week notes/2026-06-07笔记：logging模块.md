




## 一句话解释
logging是日志的意思，即用简短的话来记录项目运行时的问题

用自己的话解释这个概念是什么。

## 为什么重要
日志在我们实际的项目开发中十分重要，我们需要日志来判断项目的实际运行情况，在发现风险和错误时要进行评估和修改，保证我们做的项目是有用的而不是仅仅的一个玩具
说明它和当前项目、AI Agent、RAG、后端工程或科班基础有什么关系。


## 概念及代码
logging有5个级别
- debug：调试信息
- info：普通运行信息
- warning：警告但是文件还能运行的
- error：错误，某个功能错误
- critical：严重错误：程序快挂了

```python
import logging

logging.debug("debug msg")
logging.info("info msg")
logging.warning("warning msg")
logging.error("error msg")
logging.critical("critical msg")

#一般只有在warning及以上级别的日志信息会打印在终端上

#我们也可以手动修改打印信息的基础等级

logging.basicConfig(level = INFO)
#将打印的初始等级改成info级

INFO:root:info msg
WARNING:root:warn msg
ERROR:root:error msg
```

|级别|数值|何种含义 / 何时使用|
|---|---|---|
|logging.NOTSET[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.NOTSET "Link to this definition")|0|当在日志记录器上设置时，表示将查询上级日志记录器以确定生效的级别。 如果仍被解析为 `NOTSET`，则会记录所有事件。 在处理器上设置时，所有事件都将被处理。|
|logging.DEBUG[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.DEBUG "Link to this definition")|10|详细的信息，通常只有试图诊断问题的开发人员才会感兴趣。|
|logging.INFO[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.INFO "Link to this definition")|20|确认程序按预期运行。|
|logging.WARNING[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.WARNING "Link to this definition")|30|表明发生了意外情况，或近期有可能发生问题（例如‘磁盘空间不足’）。 软件仍会按预期工作。|
|logging.ERROR[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.ERROR "Link to this definition")|40|由于严重的问题，程序的某些功能已经不能正常执行|
|logging.CRITICAL[](https://docs.python.org/zh-cn/3.14/library/logging.html#logging.CRITICAL "Link to this definition")|50|严重的错误，表明程序已不能继续执行|
