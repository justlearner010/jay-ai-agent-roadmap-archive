

在 AI/LLM 领域里，**“mock LLM”** 指的不是一个真正生成语言的模型，而是一个 **“模拟/伪装 LLM 行为的工具或对象”**，通常用于开发和测试。

具体来说：

---

## 1. 核心概念

- **Mock**：在软件工程中，mock 是一个“假的对象”，它模仿真实对象的接口和行为，但不执行真实逻辑。

- **LLM**：大语言模型（Large Language Model），如 GPT、Claude、LLaMA 等。


所以 **mock LLM** 就是：

> 一个模拟大语言模型的对象或函数，用来测试你的程序和 LLM 的交互，而不需要真正调用 API 或模型。

---

## 2. 为什么要用 mock LLM

1. **节约成本**

    - 真正调用 GPT-4 或 Claude 可能需要计费。

    - Mock LLM 可以返回固定结果，节省 API 调用。

2. **便于单元测试**

    - 当你写函数依赖 LLM 输出时，用 mock 可以控制返回值，保证测试可预测。

3. **加快开发速度**

    - 不用每次都等待模型响应。


---

## 3. 举例（Python + unittest.mock）

假设你有一个函数：

```python
def summarize(text, llm):
    response = llm.generate(f"Summarize: {text}")
    return response
```

真实情况下 `llm.generate()` 会调用 GPT API。
测试时可以 mock：

```python
from unittest.mock import Mock

# 创建一个 mock LLM
mock_llm = Mock()
mock_llm.generate.return_value = "这是总结内容"

# 测试
text = "今天我学习了Python的argparse"
summary = summarize(text, mock_llm)

print(summary)
# 输出: 这是总结内容
```

这里 `mock_llm` 就是 **mock LLM**，它模仿真实模型，但完全在本地运行。

---

## 4. 高级用途

- 模拟不同模型输出策略（temperature, randomness）

- 测试多轮对话逻辑（多轮 prompt chaining）

- 模拟错误情况（timeout, API 报错）

- 在 RAG（检索增强生成）或 Agent 系统中，先用 mock 测试 pipeline，再换真实 LLM


---

💡 总结：

> **mock LLM = 一个假的、可控的 LLM 对象，用于开发和测试，而不实际生成文本。**
> 真正的 LLM 可以替换它，而你的代码逻辑保持不变。
