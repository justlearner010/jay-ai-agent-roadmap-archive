
### `getLogger(name)`

```python
logger = getLogger(__name__)
#python会把这个名称自动定位

```
logging中的logger本质上是产生一个logging对象，用于记录各个模块的信息，`__name__`的作用就是自动定位这个名字，保证开发者清楚问题所在的地方

### `logging.basicConfig()`

```python
logging.basicConfig(
	level = logging.INFO
	format = "%(asctime)s :%(filename)s %(message)s"
	)
	>>> 12:30 : word_chunk.py chunk created
```

易错点 ：
- basicConfig只能生效一次
- 如果根 logger 已有 handler，则 basicConfig 什么都不做。
- 不要每个模块都写basicConfig()
- 应该在main.py中统一配置好
**推荐记住的 format 字段**

你以后最常见的是：

| 字段            | 作用       |
| ------------- | -------- |
| %(asctime)s   | 时间       |
| %(name)s      | logger名称 |
| %(levelname)s | 日志等级     |
| %(message)s   | 日志内容     |
| %(filename)s  | 文件名      |
| %(lineno)d    | 行号       |
| %(funcName)s  | 函数名      |


### logging 和程序报错的区别

logging是记录程序活着时候的每一步的文件，用于监控程序的每一步，在实际工程中如果程序没有报错但是输出的结果是错的，这个时候就要看日志里的哪一步错误了，光有程序报错是不够的；

**Exception 告诉你程序死在哪里。Logging 告诉你程序活着的时候干了什么。**


### raise 、try/except/logging/raise(在except)的关系

| 功能                   | 作用      | 类比        |     |
| -------------------- | ------- | --------- | --- |
| `raise`              | 抛出异常    | 发现事故报警    |     |
| `try/except`         | 捕获异常    | 处理事故      |     |
| `logging`            | 记录信息    | 行车记录仪     |     |
| `logger.exception()` | 记录完整异常栈 | 事故录像+事故报告 |     |
| `raise`（在 except 中）  | 继续传播异常  | 上报领导      |     |
