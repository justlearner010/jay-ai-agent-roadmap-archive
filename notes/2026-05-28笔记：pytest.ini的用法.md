
### 1.`pytest.ini`

作用：
- 配置`pytest`项目级参数
- 设置Python模块搜索路径，避免在测试文件里面写`sys.path.append(..)
- 配置默认marker、参数、选项
- 保证`pytest`在工程结构下正确找到源代码和测试文件
最常用示例：
```python
[pytest]
pythonpath = .

```
- pythonpath = .可以把项目的根目录加入Python搜素路径
- 这样测试文件就可以直接`from app(folder name).text_chunk import text_chunk`
### 2.`_init_.py`

作用：
- 告诉python：这个目录是一个package（包），可以被import
- 支持模块化开发、规范化import
- 可以用来在包级别暴露接口




- `pytest.ini` → 配置 pytest，规范测试运行和模块搜索路径
- `__init__.py` → 告诉 Python 这是包，支持模块 import 与组织
