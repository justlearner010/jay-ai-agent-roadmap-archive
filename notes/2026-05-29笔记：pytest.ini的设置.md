## 假设项目结构

推荐：

```
project/│├── pytest.ini│├── main.py│└── tests/    ├── __init__.py    └── test_main.py
```

---

## 最简单的 pytest.ini

```
[pytest]pythonpath = .
```

意思：

```
把当前项目根目录加入 Python 搜索路径
```

这样测试文件里就能直接：

```
from main import remove_duplicates
```

而不用：

```
import syssys.path.append("..")
```

---

## 更规范的项目结构

很多项目会写成：

```
project/│├── pytest.ini│├── app/│   ├── __init__.py│   └── main.py│└── tests/    ├── __init__.py    └── test_main.py
```

对应：

```
[pytest]pythonpath = .
```

测试：

```
from app.main import remove_duplicates
```

---

## 指定测试目录

```
[pytest]testpaths = tests
```

意思：

```
pytest 默认只扫描 tests 文件夹
```

这样：

```
pytest
```

就等价于：

```
pytest tests
```

---

## 显示更详细结果

```
[pytest]addopts = -v
```

运行：

```
pytest
```

自动变成：

```
pytest -v
```

输出：

```
test_main.py::test_remove_duplicates PASSED
```

---

## 我自己常用的配置

```
[pytest]pythonpath = .testpaths = testsaddopts = -v
```

### `__init__.py` 又是什么？

当一个文件夹里有：

```
app/├── __init__.py└── main.py
```

Python 会把 `app` 视为一个包（package）。

于是可以：

```
from app.main import hello
```

而不是：

```
import main
```

记忆：

|文件|作用|
|---|---|
|pytest.ini|配置 pytest|
|**init**.py|告诉 Python 这是一个包|
|test_xxx.py|pytest 自动识别的测试文件|
|assert|判断结果是否正确|

对于你准备做 AI Agent 实习的目标来说，建议从现在开始养成这个结构：

```
project/├── pytest.ini├── src/├── tests/└── README.md
```
