
## 1. README 是什么？

README 是项目的“说明书”。

当别人打开你的 GitHub 仓库时，最先看到的通常就是 README，所以它承担着介绍项目的职责。

一个好的 README 通常回答这些问题：

- 项目是干什么的（What）
- 为什么有用（Why）
- 如何开始使用（How）
- 遇到问题去哪求助（Help）
- 谁在维护项目（Maintainers）

例如：

```
My Calculator一个用 Python 编写的计算器。功能：- 加减乘除- 幂运算安装：pip install ...使用：python main.py
```

---

## 2. README 放在哪里？

GitHub 会自动识别以下位置的 README：

```
.github/README.mdREADME.mddocs/README.md
```

优先级：

```
.github/   ↓仓库根目录   ↓docs/
```

如果三个地方都有 README：

```
.github/README.md
```

会被优先显示。

---

## 3. README 太大会怎样？

GitHub 会显示 README，

但超过：

```
500 KiB
```

的部分会被截断（truncated）。

所以 README 应该保持简洁。

---

## 4. Profile README（个人主页 README）

这是 GitHub 一个很有意思的功能。

如果你创建一个仓库：

```
用户名 = 仓库名
```

例如：

```
用户名：krisliu仓库：krisliu
```

并在里面写 README，

那么 README 会直接显示在你的 GitHub 主页。

很多程序员都会用它展示：

- 技能栈
- 项目
- 联系方式
- GitHub Stats

例如：

```
# Hi, I'm KrisComputer Science StudentCurrently learning:- Python- Data Structures- AI Agents
```

---

## 5. 自动生成目录（Table of Contents）

如果 README 有很多标题：

```
# Project## Installation## Usage## Features## Contributing
```

GitHub 会自动生成：

```
Outline├─ Installation├─ Usage├─ Features└─ Contributing
```

方便用户跳转。

---

## 6. 标题链接（Section Links）

例如：

```
## Installation
```

GitHub 会自动生成一个锚点链接。

你可以直接链接到这一部分：

```
https://github.com/...#installation
```

点击标题旁边的小链条图标即可获得链接。

---

## 7. 相对路径链接（Relative Links）

这是这一页比较重要的技术内容。

假设项目结构：

```
project/│├─ README.md│└─ docs/   └─ CONTRIBUTING.md
```

在 README 中写：

```
[Contribution Guide](docs/CONTRIBUTING.md)
```

GitHub 会自动跳转到：

```
docs/CONTRIBUTING.md
```

这种叫：

**相对路径（Relative Path）**

---

### 为什么推荐相对路径？

不要这样：

```
https://github.com/xxx/project/blob/main/docs/CONTRIBUTING.md
```

因为：

- 仓库改名可能失效
- Fork 后可能失效
- Clone 到本地后无法点击

而相对路径：

```
docs/CONTRIBUTING.md
```

无论：

- 本地
- GitHub
- Fork

都能正常工作。

---

## 8. 图片也支持相对路径

例如：

```
![logo](images/logo.png)
```

项目结构：

```
project/│├─ README.md│└─ images/    └─ logo.png
```

GitHub 会自动找到图片。

---

## 9. README 不要写太长

GitHub 建议：

README 只写：

- 项目介绍
- 安装
- 快速开始
- 贡献方式

对于详细文档：

```
WikiDocsDocumentation
```

更合适。

例如：

```
README    ↓快速上手Docs/    ↓详细教程
```
