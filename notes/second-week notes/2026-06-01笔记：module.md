

# 1. import 到底干了什么

很多初学者以为：

```python
import math
```

是把 `math` 里的代码复制过来。

实际上是：

1. Python 找到模块

2. 执行模块中的代码（第一次导入时）

3. 创建模块对象

4. 将模块名绑定到当前命名空间


例如：

```python
import math

print(math.pi)
```

这里 `math` 是一个对象。

---

# 2. import 和 from import 的区别

### 推荐

```python
import math

math.sqrt(16)
```

优点：

- 看得出函数来自哪里

- 不容易命名冲突


---

### 直接导入名称

```python
from math import sqrt

sqrt(16)
```

优点：

- 少打字


缺点：

- 不知道 sqrt 来自哪个模块


---

### 尽量避免

```python
from math import *
```

因为：

```python
from math import *
from numpy import *
```

很容易覆盖同名函数。

面试和实际开发里基本不用。

---

# 3. **name** 是什么

这是本章最重要的内容之一。

当文件直接运行：

```bash
python main.py
```

则：

```python
__name__ == "__main__"
```

---

当文件被导入：

```python
import main
```

则：

```python
__name__ == "main"
```

---

因此经常看到：

```python
def main():
    print("hello")

if __name__ == "__main__":
    main()
```

作用：

- 被导入时不执行

- 直接运行时执行


以后几乎所有项目都会这么写。

---

# 4. 模块只会导入一次

例如：

```python
# test.py

print("loaded")
```

```python
import test
import test
import test
```

输出：

```text
loaded
```

只会执行一次。

Python 会缓存导入结果。

缓存位置：

```python
sys.modules
```

---

# 5. 模块搜索路径

这是最容易踩坑的。

Python 导入模块时会去：

```python
sys.path
```

里找。

查看：

```python
import sys

print(sys.path)
```

通常包括：

- 当前目录

- 标准库目录

- site-packages


---

例如：

```python
project/
│
├── main.py
└── utils.py
```

可以：

```python
import utils
```

因为当前目录就在搜索路径里。

---

# 6. 不要给自己的文件起标准库名字

这是经典错误。

例如：

```text
random.py
```

然后：

```python
import random
```

导入的是你自己的文件。

而不是标准库：

```python
import random

random.randint(...)
```

直接报错。

同理：

```text
math.py
json.py
sys.py
string.py
```

都不要起。

# 7. dir() 调试神器

查看模块有什么内容：

```python
import math

dir(math)
```

输出：

```python
[
'sin',
'cos',
'tan',
'sqrt',
'pi'
...
]
```

不会用模块时：

```python
dir(模块名)
```

先看一眼。

---

# 8. 包（Package）是什么

模块：

```text
utils.py
```

一个文件。

---

包：

```text
project/
│
├── __init__.py
├── utils.py
├── models.py
└── database.py
```

一个目录。

---

导入：

```python
from project import utils
```

或者：

```python
import project.utils
```

以后学 Flask、Django、FastAPI 时会大量出现。

---

# 9. **pycache** 不用管

你会发现：

```text
__pycache__/
```

自动生成。

里面是：

```text
xxx.pyc
```

字节码缓存。

作用：

- 下次导入更快


不用删。

不用提交到 Git。

通常：

```gitignore
__pycache__/
```

# 对你目前阶段最重要的内容

按照重要程度排序：

1. `import module`

2. `from module import name`

3. `import module as alias`

4. `if __name__ == "__main__":`

5. `sys.path`

6. 包（package）

7. `dir()`

8. `__pycache__`


如果你准备继续学 CS50P 后面的 File I/O、Unit Test、面向对象，那么一定要彻底理解：

```python
if __name__ == "__main__":
```

和

```python
import xxx
```

因为后面写测试时，你会经常遇到：

```python
# calculator.py
def add(a, b):
    return a + b

if __name__ == "__main__":
    print(add(1, 2))
```

```python
# test_calculator.py
from calculator import add
```

这正是模块这一章最核心的实际应用。
