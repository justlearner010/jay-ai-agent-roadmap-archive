# 一句话总结

对于需要发布、测试、长期维护的 Python 项目，官方更推荐 **src layout**，因为它能避免很多隐藏的导入（import）问题。

---

# 两种目录结构是什么

## 1. Flat Layout（平铺结构）

```
project/├── pyproject.toml├── README.md├── setup.py├── awesome_package/│   ├── __init__.py│   └── module.py└── tests/
```

代码包直接放在项目根目录。

导入：

```
import awesome_package
```

---

## 2. Src Layout（推荐）

```
project/
├── pyproject.toml
├── README.md
├── setup.py
├── src/
│   └── awesome_package/
│       ├── __init__.py
│       └── module.py
└── tests/
```

代码放在：

```
src/awesome_package/
```

里面。

---

# 为什么会有 src layout

核心原因：

> Python 会自动把当前工作目录加入 sys.path。

例如：

```
import sysprint(sys.path[0])
```

输出：

```
当前目录
```

因此：

```
project/├── awesome_package/
```

即使你没有安装项目：

```
pip install .
```

下面依然能运行：

```
import awesome_package
```

因为 Python 从当前目录找到了它。

---

# 问题在哪里

假设：

```
project/├── setup.py├── awesome_package/
```

你写：

```
pytest
```

测试通过。

于是你觉得：

> 我的包没问题。

但实际上：

你测试的是

```
当前目录里的代码
```

而不是

```
安装后的代码
```

---

# 一个真实的坑

假设：

```
setup.py
```

配置错误：

```
packages=["wrong_name"]
```

导致发布到 PyPI 时：

```
pip install my_package
```

安装出来是空包。

但是本地：

```
import awesome_package
```

还能正常运行。

因为导入的是：

```
./awesome_package
```

而不是：

```
site-packages/awesome_package
```

结果：

```
本地没问题上线炸了
```

---

# src layout 如何解决

结构变成：

```
project/├── src/│   └── awesome_package/
```

此时：

```
import awesome_package
```

会失败：

```
ModuleNotFoundError
```

因为：

```
project/
```

已经不包含：

```
awesome_package/
```

了。

必须：

```
pip install -e .
```

之后才能导入。

这样你开发时：

```
导入的永远是安装后的包
```

而不是源代码目录里的包。

---

# 官方认为最大的优点

## 防止误导入

Flat Layout：

```
project/├── awesome_package/
```

Python：

```
import awesome_package
```

可能导入：

```
当前目录
```

版本。

---

Src Layout：

```
project/├── src/│   └── awesome_package/
```

必须安装。

因此：

```
开发环境=真实用户环境
```

更接近。

---

# 第二个优点

避免把乱七八糟文件加入 import 路径。

Flat Layout：

```
project/├── README.md├── setup.py├── noxfile.py├── tox.ini
```

如果 editable install：

```
pip install -e .
```

某些实现方式会把整个：

```
project/
```

加入：

```
sys.path
```

于是：

```
import setup
```

居然能成功。

```
import noxfile
```

也能成功。

但发布后：

```
这些文件根本不存在
```

程序就炸了。

---

Src Layout：

```
project/src/└── awesome_package/
```

加入路径的是：

```
src/
```

不会把：

```
README.mdsetup.pynoxfile.py
```

暴露给 import 系统。

更安全。

---

# 缺点是什么

需要先安装。

Flat Layout：

```
python script.py
```

直接跑。

---

Src Layout：

通常需要：

```
pip install -e .
```

或者：

```
python -m package
```

先安装。

开发步骤多一步。

---

# 为什么现在很多项目都用 src

比如：

- pytest
- requests
- black
- pydantic
- pandas
- scipy

很多现代项目都采用：

```
src/
```

结构。

原因是：

```
大型项目=工程可靠性>少打一条命令
```

---

# 第一性原理理解

这个页面本质只在解决一个问题：

> 如何确保开发时运行的代码，与用户安装后运行的代码是同一份代码？

Flat Layout：

```
开发环境 ≠ 用户环境
```

因为会优先导入当前目录。

---

Src Layout：

```
开发环境 ≈ 用户环境
```

因为必须经过安装流程。

---

对于你以后想做的 **AI Agent、后端服务、Python SDK、PyPI 包**，我建议默认采用：

```
project/├── pyproject.toml├── src/│   └── my_package/├── tests/└── README.md
```
