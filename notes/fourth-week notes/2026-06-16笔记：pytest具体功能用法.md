

可以把这部分 pytest 用法压缩成一张表：

|用法|作用|示例|
|---|---|---|
|`pytest`|运行当前目录下所有测试|`pytest`|
|`pytest 文件名`|只运行某个测试文件|`pytest tests/test_summarize.py`|
|`pytest 目录名`|只运行某个目录下的测试|`pytest tests/`|
|`::`|精确运行某个函数、类、方法|`pytest tests/test_summarize.py::test_summarize_brief`|
|`-k`|按测试名称筛选|`pytest -k summarize`|
|`-m`|按 marker 标签筛选|`pytest -m slow`|
|`@pytest.mark.xxx`|给测试贴标签|`@pytest.mark.slow`|
|`-q`|简洁输出|`pytest -q`|
|`-x`|遇到第一个失败就停止|`pytest -x`|
|`--durations=N`|查看最慢的 N 个测试|`pytest --durations=10`|
|`python -m pytest`|用 Python 模块方式运行 pytest|`python -m pytest`|
|`--fixtures`|查看可用 fixture|`pytest --fixtures`|
|`--help`|查看 pytest 命令帮助|`pytest --help`|

## 重点区别

### `-k`：按名字筛选

```bash
pytest -k summarize
```

会运行名字里包含 `summarize` 的测试，例如：

```python
def test_summarize_brief():
    ...
```

适合快速找某一类函数相关测试。

---

### `-m`：按标签筛选

```bash
pytest -m slow
```

会运行贴了 `slow` 标签的测试：

```python
@pytest.mark.slow
def test_large_file():
    ...
```

适合区分：

```text
slow        慢测试
network     需要联网
integration 集成测试
unit        单元测试
```

---

## 最常用组合

平时开发时：

```bash
pytest -q
```

调试某个失败测试：

```bash
pytest tests/test_summarize.py::test_summarize_brief
```

只跑某类测试：

```bash
pytest -k summarize
```

跳过慢测试：

```bash
pytest -m "not slow"
```

遇到失败就停：

```bash
pytest -x
```

## 你现在最该掌握的核心

现阶段不用记太多，掌握这四个就够：

```bash
pytest
pytest 文件名
pytest 文件名::函数名
pytest -k 关键词
```

再往后项目变大了，再用：

```bash
pytest -m 标签
pytest --durations=10
```
