# 前 90 天计划：2026-05-25 到 2026-08-16

目标：建立学习节奏，补强 Python/TypeScript 工程基础，并完成第一个 AI 学习助手原型。

逐周执行细节见：`weekly-plans/first-12-weeks.md`。

## 第 1-2 周：环境搭建与 Python 复习

产出目标：

- 安装并验证 Python、Node.js、Git、VS Code 或 Cursor。
- 创建一个 GitHub 学习仓库。
- 复习 Python：函数、类、文件、异常、类型标注。
- 完成 20 个 Python 小练习，其中至少 5 个带 pytest 测试。

交付物：

- `python-foundations` 文件夹，包含练习、测试和学习说明。

## 第 3-4 周：Python 工程基础

产出目标：

- 学会虚拟环境和包管理。
- 学会 pytest fixture、参数化测试和基础 mock。
- 做一个命令行文本总结器，先用 mock LLM 接口。
- 练习日志和错误处理。

交付物：

- CLI 小工具：输入文本文件，输出总结，并保存运行日志。

## 第 5-6 周：TypeScript 与 Next.js 基础

产出目标：

- 学 TypeScript 类型、interface、异步函数、fetch。
- 做一个简单 Next.js 页面：文本输入、输出区域、loading 状态、错误状态。
- 加一个 API route，返回 mock 总结结果。

交付物：

- 一个带前端界面的 AI 学习助手雏形，后端先用 mock 数据。

## 第 7-8 周：接入真实 LLM API

产出目标：

- 用环境变量管理 API key。
- 从后端调用一个 LLM API。
- 返回结构化结果：总结、要点、追问建议。
- 在日志里记录基础成本和调用情况。

交付物：

- AI 学习助手可以通过真实 LLM 总结文本。

## 第 9-10 周：问答与历史记录

产出目标：

- 支持基于当前文本提问。
- 保存会话历史，可以用本地文件或 SQLite。
- 增加历史记录页面。
- 为后端核心逻辑写 pytest。

交付物：

- AI 学习助手支持粘贴/上传、总结、提问、历史记录。

## 第 11-12 周：打磨与第一次作品集复盘

产出目标：

- 写一份清楚的 README。
- 加架构图。
- 补失败案例：空输入、超长文本、API 失败、无关问题。
- 录一个 2-3 分钟 demo 视频。
- 写 3 条这个项目的简历 bullet。

交付物：

- 一个可以放到 GitHub 展示的项目：README、截图或 demo、测试、后续计划都齐全。
