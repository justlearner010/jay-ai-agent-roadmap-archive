## 下周 Next.js 页面需要的 API 数据结构

下周先做 mock API，不急着接真实 LLM。前端页面只需要关心三件事：

- 用户输入的文本。
- 用户选择的总结模式。
- API 返回 summary 或错误信息。

### `POST /api/summarize`

请求体：

```json
{
  "text": "Artificial intelligence is changing how people learn.",
  "mode": "brief"
}
```

字段说明：

- `text`：用户输入的原文，类型是 string，不能为空。
- `mode`：总结模式，先支持 `brief` 和 `bullets`。

成功响应：

```json
{
  "summary": "Brief summary: this text has 7 words",
  "mode": "brief",
  "inputChars": 54
}
```

错误响应：

```json
{
  "error": "Input text is empty.",
  "code": "EMPTY_INPUT"
}
```

前端页面状态：

- `idle`：还没有提交。
- `loading`：正在请求 API。
- `success`：展示 summary。
- `error`：展示错误信息。

下周第一版 TypeScript 类型可以先写成：

```ts
type SummaryMode = "brief" | "bullets";

type SummarizeRequest = {
  text: string;
  mode: SummaryMode;
};

type SummarizeSuccessResponse = {
  summary: string;
  mode: SummaryMode;
  inputChars: number;
};

type SummarizeErrorResponse = {
  error: string;
  code: "EMPTY_INPUT" | "INVALID_MODE" | "SERVER_ERROR";
};
```
