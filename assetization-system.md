# 学习资产化系统

目标：不追仓库数、文章数或刷题数，而是把同一次学习持续转化为可复用、可展示、可求职的 AI Agent 工程资产。

## 资产链

```text
概念理解
→ 可运行代码
→ 测试/运行证据
→ 项目功能
→ README/demo/失败案例
→ 简历 bullet + 5 分钟面试故事
```

笔记只是原材料。笔记中的知识进入代码、测试、项目或求职材料后，才完成资产转化。

## 计数规则

- 单个知识点不创建新仓库；练习优先汇入 AI 学习助手或后续作品集项目。
- 每天至少留一个证据：测试输出、运行记录、截图、代码 diff 或真实 commit。
- 每周只设一个主交付物；其他学习必须解决它的实际问题。
- 每周最多读一个开源项目。必须记录一个迁移点并实际迁移，否则不计为成果。
- AI 生成代码只有在能解释、能测试、能修改后才计入个人资产。
- 公开内容只服务项目：README、失败案例和项目复盘；不设博客数量目标。
- Fork、收藏、未运行的教程代码和未迁移的笔记不计入交付物。

## 发布节奏

- 每 2 周：一个技术里程碑，至少包含集成功能、测试/运行证据和 README 变更。
- 每 4 周：一个可复现版本，补齐截图或 demo、失败案例、架构图、简历 bullet 和 5 分钟讲解纲要。
- 日常可以小步 commit，但仅在里程碑验收后对外强调发布。

## 每周验收

一周只有同时满足下列条件才算合格：

- 一个功能进入当前主项目。
- 至少两个测试或运行证据。
- 上一轮必改项已关闭，或明确记录了阻塞原因。
- 有真实 commit，或在周复盘说明未提交原因。
- `trackers/weekly-log.csv` 和 `trackers/project-scorecard.csv` 中的证据、状态和下一步与当前文件一致。

## 路线边界

- 保持 70% AI Agent/RAG/LLM 应用工程主线和 30% Java 后端兼容线。
- 当前不增加 AI Infra、CUDA、vLLM 或新项目支线。
- 算法、科班基础和开源阅读只按当前项目卡点嵌入，不替代主交付物。

## 配套文件

- `weekly-operating-system.md`：20 小时日历和周执行节奏。
- `github-portfolio-checklist.md`：4 周可复现版本的发布门槛。
- `templates/project-milestone-review.md`：技术里程碑和可复现版本的统一复盘模板。
- `trackers/weekly-log.csv`：每周实际时间、成果链接和复盘。
- `trackers/project-scorecard.csv`：项目资产完整度。
