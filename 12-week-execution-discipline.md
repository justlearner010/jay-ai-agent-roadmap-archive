# 前 12 周执行纪律

目标：不再增加学习科目，而是提高完成质量。第一阶段的重点是先建立“能运行、能解释、能记录”的闭环，再逐步过渡到 Python 工程习惯、测试习惯和 GitHub 证据链。

## 每天最低要求

每天至少完成下面 3 项中的 2 项：

- 一个可运行的小功能。
- 一篇学习日志。
- 一次 Git commit。

理想状态是每天完成完整闭环：

- 1 个小功能。
- 1-2 个运行证据；学到测试后再逐步加入 pytest。
- 1 篇 10-20 行学习日志。
- 1 次 commit。

## 每周硬性要求

每周必须完成：

- 至少一次 Codex 作业批改。
- 至少一次根据批改意见进行修改。
- 至少 2 个运行证据。第 2 周开始再逐步要求 pytest。
- 至少一次 README、学习笔记或项目说明更新。
- 周日晚复盘，并更新 `trackers/weekly-log.csv`。

## 分阶段提升标准

### 第一次完成：入门练习

一个新知识点刚开始练时，先达到：

- 代码能运行。
- 你能解释每一段主要代码。
- 结果大体正确。
- 有学习日志和运行证据。

这个阶段不要求一次写成函数、测试、CLI 或 README。

### 第二次修改：小步改进

读懂第一版后，再做 1-2 个改进：

- 修明显 bug。
- 改清楚变量名。
- 减少重复。
- 可以尝试写一个简单函数。
- 补一个边界情况。

### 后续整理：工程化版本

当一个练习已经能解释清楚，再逐步升级为可复用模块。

## 每个练习的最终标准

每个 Python 练习最终可以达到：

- 有函数封装。
- 不依赖 `input()` 才能测试。
- 核心逻辑返回值清楚，优先返回 dict、list 或字符串。
- 可以被 pytest 调用。
- 能处理至少一个边界情况。
- 你能解释它对应的科班基础。

例如 `txt_check.py` 后续可以升级成：

- 文件名：`text_stats.py`。
- 核心函数：`count_text_stats(path)`。
- 返回结果：`{"lines": 3, "non_empty_lines": 2, "words": 10, "characters": 50}`。
- 测试文件：`tests/test_text_stats.py`。

## 批改和修改节奏

不要只追新任务。每天开始前先看昨天批改意见：

1. 有必改项：先修必改项。
2. 必改项修完：再做当天新功能。
3. 当天结束：提交作业批改。
4. 复查通过：再推送 GitHub。

如果连续两天没有完成必改项，第三天暂停新知识，只做修正。

## 不增加的内容

前 12 周先不要主动扩展这些内容：

- Java / Spring Boot。
- 数据库系统深入。
- LangGraph。
- RAG 完整系统。
- Docker 深入。
- 大量算法刷题。

它们不是不重要，而是还没到当前阶段。现在先把 Python 工程、测试、CLI、Next.js、API 和 LLM 接入做扎实。

## 每周自检问题

周日复盘时回答：

1. 本周哪个代码文件从“脚本”升级成了“可测试模块”？
2. 本周至少 2 个测试在哪里？
3. 本周哪条批改意见已经修完？
4. 本周 GitHub 上有什么证据？
5. 下周要减少什么无效学习？
