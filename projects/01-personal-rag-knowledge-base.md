# 项目 1：个人 RAG 知识库

## 目标

做一个以后端为主的 RAG 系统，可以导入文档，并基于文档内容回答问题，同时给出来源引用。

## 必做功能

- 上传或导入文本、Markdown 或 PDF 文档。
- 把文档切成带 metadata 的文本块。
- 生成 embedding，并存入 Chroma 或 FAISS。
- 根据用户问题检索相关文本块。
- 基于检索结果生成有依据的回答。
- 返回引用信息：文档名和文本块位置。
- 自建 20-50 个测试问题，形成基础评估集。

## 推荐技术栈

- Python
- FastAPI
- Pydantic
- Chroma 作为 v1 向量库
- pytest
- 任一主流 LLM API

## 验收标准

- API 可以导入至少 5 份文档。
- API 可以回答问题并返回来源引用。
- 空输入、无关问题、超长输入都有可控处理。
- 评估脚本能输出检索命中率和回答质量备注。
- README 说明架构、安装、API 示例和已知失败案例。
