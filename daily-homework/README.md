# 每日作业要求与推荐阅读

这个目录把前 12 周计划拆成每天可以直接执行的作业。每个每日文件都包含：

- 今日目标
- 作业要求
- 推荐阅读材料
- 提交物
- 验收标准
- 批改重点

使用方式：

1. 每天先打开对应 `week-xx/day-xx.md`。
2. 先完成“作业要求”，再看“推荐阅读材料”。阅读服务于当天作业，不单独囤课。
3. 当天结束时，把提交物整理到学习日志和作业提交模板里。
4. 对 Codex 说：`codex，批改今天作业`。
5. 第二天优先处理批改里的必改项。

## 执行原则

- 每天只追一个主目标，不把后期 RAG/Agent 标准提前压到 Python 入门阶段。
- 每周至少留下一次可批改证据：代码、测试、日志、截图或 demo。
- 推荐阅读材料放在作业要求下方，是为了边做边查；如果阅读和作业冲突，优先完成作业。
- 每周日必须写复盘，并更新 `trackers/weekly-log.csv`。
