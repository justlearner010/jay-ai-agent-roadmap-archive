# 每日作业要求与推荐阅读

这个目录把前 12 周计划拆成每天可以直接执行的作业。每个每日文件都包含：

- 今日目标
- 作业要求
- 推荐阅读材料
- 提交物
- 验收标准
- 批改重点

使用方式：

1. 每天先打开对应 `week-xx/day-xx.md`。
2. 先完成“作业要求”，再看“推荐阅读材料”。阅读服务于当天作业，不单独囤课。
3. 当天结束时，学习日志只写今日三行；详细运行结果和证据放到作业提交模板里。
4. 对 Codex 说：`codex，批改今天作业`。
5. 第二天优先处理批改里的必改项。

## 执行原则

- 每天只追一个主目标，不把后期 RAG/Agent 标准提前压到 Python 入门阶段。
- 每天优先完成最小闭环：一个主线动作、一个证据、三行学习日志。
- 第 1-4 周不加 Java，先跑通 Python、pytest、CLI 和 GitHub 证据链。
- 第 5-8 周每周只安排 2 个 Java 副线触点，目标是 Java 基础、Maven、JUnit 和最小 Spring Boot API。
- 第 9-12 周每周安排 3 个 Java 后端触点，目标是 CRUD、数据库、任务模型和 Java + AI 服务调用雏形。
- Java 副线不要求每天推进；当天主线任务已经吃紧时，优先保证 AI 学习助手主线完成。
- Java 副线必须从已有 Python / AI 学习助手任务过渡：优先重写、封装、测试或网关化已经理解的文本处理、校验、API、持久化和任务记录能力。
- 科班基础嵌入使用 `resources/cs-foundations-side-track.md` 的固定小格：今日基础点、对应项目任务、最小验收。
- 每天安排 20-30 分钟算法微练习，优先 1 道 Easy；超过 30 分钟还没做完就停止，记录思路、卡点和对应数据结构。
- 当天主线任务很重时，算法微练习可以降级为读题、写输入输出和伪代码，不强求 AC。
- 每天可以保留一个可选自由探索格，20-40 分钟，只追一个好奇问题；它不要求代码产出，只要求记录问题和发现。
- 自由探索不能替代当天主线交付；如果主线已经超时，探索格可以跳过，不需要补债。
- 如果连续两天主线或必改项没有完成，第三天自动删掉算法 AC、长笔记、额外阅读和自由探索，只做主线修正。
- 每周至少留下一次可批改证据：代码、测试、日志、截图或 demo。
- 推荐阅读材料放在作业要求下方，是为了边做边查；如果阅读和作业冲突，优先完成作业。
- 每周日必须写复盘，并更新 `trackers/weekly-log.csv`。
