# 第 12 周：第一次作品集复盘

> 定位：滚动 4 周之外的候选任务库，不绑定日期；只有被周主问题选中后才进入计划。

主交付物：AI 学习助手 GitHub 展示版；Java + AI 后端网关形成最小 demo 说明。

科班基础嵌入：项目讲解、简历表达、从文本问答过渡到 RAG 的数据流理解；把同一项目分别过渡成 AI Agent 叙事和 Java 后端叙事。

## 本周源码阅读

主线阅读：阅读 [langchain-ai/rag-from-scratch](https://github.com/langchain-ai/rag-from-scratch)。

只看：

- indexing、retrieval、generation 的整体顺序。
- 一个最容易看懂的 notebook 小节。

迁移点：把下一阶段个人 RAG 知识库需求写成 `文档导入 -> chunking -> embedding -> retrieval -> answer` 的流程。

Java 后端对照：回看 [spring-guides/gs-rest-service](https://github.com/spring-guides/gs-rest-service) 或 [spring-projects/spring-petclinic](https://github.com/spring-projects/spring-petclinic)。

只看：

- REST API 如何对外暴露资源。
- README 如何说明运行和请求示例。

迁移点：写清 Java + AI 后端网关负责业务 API，Python RAG 服务负责 AI 推理。
